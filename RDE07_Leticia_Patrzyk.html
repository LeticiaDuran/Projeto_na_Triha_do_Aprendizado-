Branches

São ramificações de um projeto de software ou documento e tem como objetivo o desenvolvimento paralelo de:

  • Novas funcionalidades;
  • Correção de problemas;
  • Alteração de código.

Observando a imagem acima, as bolinhas amarelas, chamadas de master ou main são o braço principal, onde os arquivos ficam, já as bolinhas verdes são chamadas de branches e podem existir várias delas, são onde os programadores podem fazer alterações sem mecher no código original diretamente, onde só depois de terem certeza que está tudo ok para a inserção no código original fazem a merge, que nada mais é que a mesclagem das branches com a master.

Sua utilização serve por exemplo para uma empresa fazer alterações em seu site, ao mesmo tempo que os usuários utilizam, sem que os mesmos sofram consequências dessas alterações no momento que estão sendo feitas ou para um grupo de programadores mecher no código ao mesmo tempo, sem interferências.

Hospedagem em sites

A hospedagem em sites normalmente é feita de maneira paga, exceto em algumas plataformas como o GitHub Pages, onde basta criar uma conta e um repositório público para ser acessado.
